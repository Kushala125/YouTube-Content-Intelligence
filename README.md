The cleaned dataset was migrated into an **SQLite database**. Eleven targeted queries were written using Window Functions, CTEs, conditional aggregation, and custom-engineered composite metrics to isolate the structural patterns underlying virality.

### ūüĒć Query 1 ‚ÄĒ Do High-Engagement Videos Get More Views?

```sql
SELECT 
  CASE 
    WHEN comments = 0 THEN 'No Comments'
    WHEN (likes * 1.0 / comments) > 20 THEN 'High Engagement'
    ELSE 'Low Engagement'
  END AS engagement_tier,
  COUNT(*) AS total_videos,
  AVG(views) AS avg_views
FROM us_trending
GROUP BY engagement_tier;
```

> **ūüďä Finding:** High-engagement videos generate **~3√ó more views** than low-engagement ones.

> **ūüí° Insight:** Engagement is a *leading indicator* of reach ‚ÄĒ not a lagging reward. YouTube's algorithm actively amplifies content showing strong early interaction signals. The mechanism is: more comments ‚Üí algorithm boost ‚Üí more views. Not the reverse.

> **‚úÖ Action:** Prioritise hooks, storytelling, and direct calls-to-action that drive comments and likes in the first hour after publishing.

---

### ūüĒć Query 2 ‚ÄĒ Does Publish Time Affect Performance?

```sql
SELECT 
  CASE 
    WHEN CAST(strftime('%H', publish_time) AS INTEGER) BETWEEN 6 AND 12 
      THEN 'Morning (6AM‚Äď12PM)'
    WHEN CAST(strftime('%H', publish_time) AS INTEGER) BETWEEN 13 AND 18 
      THEN 'Afternoon (1PM‚Äď6PM)'
    ELSE 'Night (7PM‚Äď5AM)' 
  END AS time_of_day,
  COUNT(*) AS total_videos,
  AVG(views) AS avg_views,
  AVG(likes) AS avg_likes
FROM us_trending
GROUP BY time_of_day;
```

> **ūüďä Finding:** Morning uploads achieve significantly higher average views and likes than any other window.

> **ūüí° Insight:** Early publishing creates maximum runway within the algorithm's daily recommendation cycle. Morning content enters the discovery window with the most time to accumulate interactions before curation runs ‚ÄĒ a structural, timing-based advantage with zero creative cost.

> **‚úÖ Action:** Schedule uploads consistently in the 06:00‚Äď10:00 window for maximum algorithmic exposure.

---

### ūüĒć Query 3 ‚ÄĒ Do Longer Titles Perform Better?

```sql
SELECT 
  CASE 
    WHEN LENGTH(title) < 40            THEN 'Short  (<40 chars)'
    WHEN LENGTH(title) BETWEEN 40 AND 70 THEN 'Medium (40‚Äď70 chars)'
    ELSE                                    'Long   (>70 chars)' 
  END AS title_length_category,
  COUNT(*) AS total_videos,
  AVG(views) AS avg_views,
  AVG(likes) AS avg_likes
FROM us_trending
GROUP BY title_length_category;
```

> **ūüďä Finding:** Medium-length titles (40‚Äď70 characters) consistently outperform both short and long alternatives.

> **ūüí° Insight:** This is the **Goldilocks Zone** for title metadata. Short titles lack the context that drives a click. Long titles get truncated on mobile (the dominant viewing platform). The 40‚Äď70 character window balances SEO discoverability, mobile display, and cognitive clarity simultaneously.

> **‚úÖ Action:** Treat title length as a precision engineering decision ‚ÄĒ not an afterthought. Balance curiosity + clarity + keyword within 40‚Äď70 characters.

### ūüĒć Query 4 ‚ÄĒ Top 3 Videos Per Category (Window Function)

```sql
WITH ranked_videos AS (
  SELECT 
    category_id, title, views,
    ROW_NUMBER() OVER (
      PARTITION BY category_id 
      ORDER BY views DESC
    ) AS rank
  FROM us_trending
)
SELECT * FROM ranked_videos WHERE rank <= 3;
```

> **ūüďä Finding:** Entertainment and Music dominate top positions, but niche content regularly breaks through in Gaming, How-to, and Science categories.

> **ūüí° Insight:** Viral success is category-dependent but not category-exclusive. Smaller creators can compete by benchmarking against the top performers *within their own category* ‚ÄĒ not against platform-wide giants with structurally different advantages.

> **‚úÖ Action:** Study the structural patterns (hook type, length, title format, thumbnail style) of the top 3 videos in your specific category.

### ūüĒć Query 5 ‚ÄĒ Which Videos Outperform Their Category Average? (CTE + JOIN)

```sql
WITH category_avg AS (
  SELECT 
    category_id, 
    AVG(views) AS avg_category_views
  FROM us_trending
  GROUP BY category_id
)
SELECT 
  t.title, 
  t.category_id, 
  t.views,
  c.avg_category_views,
  (t.views - c.avg_category_views) AS performance_gap
FROM us_trending t
JOIN category_avg c ON t.category_id = c.category_id
ORDER BY performance_gap DESC
LIMIT 15;
```

> **ūüďä Finding:** A small number of videos outperform their category average by extreme margins ‚ÄĒ the performance gap distribution is itself power-law shaped.

> **ūüí° Insight:** Content performance follows a **power-law distribution within every category**. A few breakout hits drive most category-level traffic. Consistency alone cannot compete with a single well-timed viral moment.

> **‚úÖ Action:** Run deliberate creative experiments for breakout content, rather than only optimising for reliable average performance.

### ūüĒć Query 6 ‚ÄĒ Composite Virality Score Model ‚≠ź *Core Innovation*

```sql
WITH features AS (
  SELECT 
    title, views, likes, comments,
    (likes    * 1.0 / views) AS like_rate,
    (comments * 1.0 / views) AS comment_rate,
    LENGTH(title) AS title_length,
    CAST(strftime('%H', publish_time) AS INTEGER) AS publish_hour
  FROM us_trending
),
scored AS (
  SELECT *,
    (  like_rate    * 0.5
     + comment_rate * 0.3
     + CASE WHEN title_length BETWEEN 40 AND 70 THEN 0.1 ELSE 0 END
     + CASE WHEN publish_hour BETWEEN 6  AND 12  THEN 0.1 ELSE 0 END
    ) AS virality_score
  FROM features
)
SELECT title, views, virality_score
FROM scored
ORDER BY virality_score DESC LIMIT 15;
```

This query engineers a **custom Virality Score** weighting four independently validated signals: like rate (50%), comment rate (30%), title length optimisation (10%), and publishing time (10%). It is the analytical centrepiece of the project.

> **ūüďä Finding:** Engagement rate is a stronger virality predictor than raw view count. High-scoring videos frequently have moderate views but exceptional audience interaction.

> **ūüí° Insight:** Platforms optimise for *interaction efficiency*, not reach alone. Niche fan-driven content regularly outperforms mainstream content on this metric ‚ÄĒ depth of audience connection matters more than breadth of distribution.

> **‚úÖ Action:** Design content for engagement rate ‚ÄĒ not view maximisation. A video seen by 500K who comment is worth more algorithmically than one seen by 2M who scroll past.

### ūüĒć Query 7 ‚ÄĒ Psychological Overperformers: What Content Lifts Above Expectations?

```sql
WITH base AS (
  SELECT 
    title, category_id, views,
    (likes * 1.0 / NULLIF(views, 0)) 
    + (comments * 1.0 / NULLIF(views, 0)) AS virality
  FROM us_trending
),
category_avg AS (
  SELECT category_id, AVG(virality) AS avg_cat_virality
  FROM base GROUP BY category_id
)
SELECT 
  b.title, b.category_id,
  (b.virality - c.avg_cat_virality) AS lift
FROM base b
JOIN category_avg c ON b.category_id = c.category_id
ORDER BY lift DESC LIMIT 20;
```

> **ūüďä Finding:** The highest-lift overperformers share three consistent characteristics ‚ÄĒ they are **emotional**, **story-driven**, or **curiosity-based** in their primary hook.

> **ūüí° Insight:** Psychology consistently outperforms production quality. Emotional hooks ‚ÄĒ personal narratives, surprising reveals, strong stakes ‚ÄĒ generate the kind of engagement that lifts a video above its category baseline. This is a **repeatable pattern**, not creative luck.

> **‚úÖ Action:** Integrate emotional triggers and storytelling arcs at the scripting stage ‚ÄĒ not as an afterthought in the edit.

### ūüĒć Query 8 ‚ÄĒ Detecting Declining Creators (LAG Window Function)

```sql
WITH lagged AS (
  SELECT 
    channel_title, views,
    LAG(views) OVER (
      PARTITION BY channel_title 
      ORDER BY publish_time
    ) AS prev_views
  FROM us_trending
)
SELECT 
  channel_title, views, prev_views, 
  (views - prev_views) AS growth
FROM lagged
WHERE prev_views IS NOT NULL
ORDER BY growth ASC LIMIT 5;
```

> **ūüďä Finding:** Several channels show consistent, measurable view decline across consecutive uploads ‚ÄĒ a clear early signal of content fatigue or audience disengagement.

> **ūüí° Insight:** Audience attention decays without content innovation. The algorithm likely reduces recommendation exposure for channels showing declining engagement trajectories ‚ÄĒ creating a compounding negative feedback loop that is hard to reverse.

> **‚úÖ Action:** Monitor your own channel's LAG growth metric. Two consecutive declines is a trigger to refresh content strategy ‚ÄĒ not a signal to post more of the same.

### ūüĒć Query 9 ‚ÄĒ Early Viral Signal Detection

```sql
WITH base AS (
  SELECT title, category_id, views,
    (likes * 1.0 / views + comments * 1.0 / views) AS engagement
  FROM us_trending
),
category_avg AS (
  SELECT category_id, AVG(engagement) AS avg_engagement
  FROM base GROUP BY category_id
),
scored AS (
  SELECT b.title, b.views,
    (b.engagement / c.avg_engagement) 
    * (1.0 / LOG(b.views + 1)) AS early_viral_score
  FROM base b
  JOIN category_avg c ON b.category_id = c.category_id
)
SELECT * FROM scored
WHERE views < 500000
ORDER BY early_viral_score DESC LIMIT 10;
```

> **ūüďä Finding:** High relative engagement combined with low absolute view count is a consistent early viral signal ‚ÄĒ identifying likely breakout videos before they scale.

> **ūüí° Insight:** This query models how recommendation engines identify viral content for amplification. A video at 50K views with 10√ó its category engagement ratio is almost certainly being tested for a broader push. The signal is detectable *before* scale happens.

> **‚úÖ Action:** For creators ‚ÄĒ maximise engagement in the first 2 hours. For analysts ‚ÄĒ this metric can power a live viral candidate detection dashboard.

---

### ūüĒć Query 10 ‚ÄĒ Are the Top 10% Fundamentally Different? (NTILE)

```sql
WITH ranked AS (
  SELECT views,
    NTILE(10) OVER (ORDER BY views DESC) AS decile
  FROM us_trending
)
SELECT 
  CASE WHEN decile = 1 THEN 'Top 10%' ELSE 'Bottom 90%' END AS group_type,
  AVG(views) AS avg_views
FROM ranked
GROUP BY group_type;
```

> **ūüďä Finding:** The top 10% of trending videos generate approximately **28√ó more views** than the bottom 90%.

> **ūüí° Insight:** Platform traffic is overwhelmingly concentrated in a tiny percentage of breakout hits. Long-tail content contributes relatively little to aggregate engagement. The platform's growth depends on viral moments ‚ÄĒ not a healthy distribution of consistent performers.

> **‚úÖ Action:** Allocate creative resources asymmetrically ‚ÄĒ invest more in content with genuine breakout potential rather than spreading effort uniformly across average-output volume.

---

### ūüĒć Query 11 ‚ÄĒ Full Content Funnel (CTE + NTILE + Conditional Aggregation)

```sql
WITH base AS (
  SELECT views,
    (likes * 1.0 / views + comments * 1.0 / views) AS engagement
  FROM us_trending
),
ranked AS (
  SELECT *, NTILE(10) OVER (ORDER BY views DESC) AS decile 
  FROM base
)
SELECT 
  COUNT(*) AS total_videos,
  SUM(CASE WHEN views > 100000    THEN 1 ELSE 0 END) AS reached_100k,
  SUM(CASE WHEN engagement > 0.05 THEN 1 ELSE 0 END) AS highly_engaged,
  SUM(CASE WHEN decile = 1        THEN 1 ELSE 0 END) AS viral_tier
FROM ranked;
```

> **ūüďä Finding:** The largest proportional drop-off occurs at the **engagement threshold** ‚ÄĒ far more videos achieve 100K views than achieve meaningful audience interaction rates.

> **ūüí° Insight:** Reach is not the bottleneck ‚ÄĒ engagement conversion is. The algorithm uses engagement signals to decide which high-reach videos deserve promotion to viral scale. Videos that fail the engagement threshold get stuck at moderate reach regardless of view count.

> **‚úÖ Action:** Optimise for interaction prompts, not just distribution. Every video needs a hook that makes engagement feel effortless and natural for the audience.

# ūüŹÜ The Three Golden Rules of YouTube Trending

*Validated in Python ¬∑ Confirmed in SQL ¬∑ Displayed in Tableau*

</div>

---

<br/>

<div align="center">

## ūüĆÖ Rule 1 ‚ÄĒ The Morning Window

</div>

> Videos published between **06:00 and 12:00** achieve approximately **3√ó the view velocity** of videos published at night.

Alignment with the start of the US workday provides the initial momentum required for the algorithm to take notice. Morning uploads enter the recommendation cycle with maximum runway. This is a **structural, free advantage** that requires zero creative effort to implement ‚ÄĒ it is purely a scheduling decision.

<br/>

<div align="center">

## ūüďĚ Rule 2 ‚ÄĒ The Title Geometry

</div>

> Titles in the **40‚Äď70 character range** consistently outperform both shorter and longer alternatives across every content category.

This is the precision window where three forces align: SEO keyword density, mobile display completeness, and cognitive clarity. Too short ‚ÄĒ the title lacks the context that drives a click. Too long ‚ÄĒ it gets truncated on mobile, the dominant viewing platform. **The 40‚Äď70 character Goldilocks Zone is the data's most actionable finding.**

<br/>

<div align="center">

## ūüí¨ Rule 3 ‚ÄĒ The Engagement Catalyst

</div>

> Comments are a stronger predictor of sustained trending status than likes. The **Engagement Signature** ‚ÄĒ a high likes-to-comments ratio ‚ÄĒ is the most reliable metric for predicting which videos stay on the trending list across multiple days.

Active engagement (comments) signals a genuine conversation ignited by the content. Passive engagement (likes) signals approval. The algorithm rewards the former far more heavily ‚ÄĒ because conversation drives session depth, which drives the ad revenue YouTube fundamentally optimises for.

## ‚ö†ÔłŹ Challenges & Solutions

<br/>

**ūüí¨ The Disabled Comments Problem**

Many trending videos had comments turned off ‚ÄĒ threatening to distort the entire engagement model.

*Solution:* SQL conditional logic isolated these as a dedicated control group, allowing the Engagement Signature to remain statistically sound while preserving their view and like data for other analyses.

<br/>

**ūüďČ The Celebrity Outlier Distortion**

High-profile uploads introduced extreme outliers that made normal content performance look like failure against simple averages.

*Solution:* NTILE-based Decile Analysis normalised the distribution, creating benchmarks applicable across all creator tiers ‚ÄĒ not just the top 1%.

<br/>

**ūüĆź Timezone Synchronisation**

Publishing timestamps originated globally, but all performance benchmarking needed to align to US Eastern Time (the dominant audience timezone).

*Solution:* Python's `pytz` library built a transformation pipeline that re-anchored all timestamps ‚ÄĒ ensuring the morning advantage finding reflected real audience behaviour, not a timezone artefact.
